# Slides 33–38 — Technical Deep Dives

## Slide 33 — Capability Deep Dives

- Drawn from Microsoft Learn public documentation.
- Attach relevant deep dives to modules for architect-level audiences.

## Slide 34 — OneLake Shortcuts: Virtualize Without Copying

- Objects in OneLake point to other storage locations like symbolic links.
- Supported sources include ADLS Gen2, Azure Blob, Amazon S3, Google Cloud Storage, Dataverse, Apache Iceberg, OneDrive, SharePoint, and on-premises via OPDG.
- Shortcuts appear as folders and can be queried by Fabric engines.
- Cross-cloud reads are cached with 1–28 day retention.
- Up to 100,000 shortcuts per item.

## Slide 35 — Mirroring in Fabric

- Fully managed continuous replication into OneLake in Delta format.
- Supports database, metadata, and open mirroring.
- Includes Azure SQL, SQL Server, Cosmos DB, PostgreSQL/MySQL preview, Snowflake, Databricks, BigQuery preview, Oracle, SAP, Dremio preview, and Fabric SQL database.
- Free tier: 1 TB mirroring storage per capacity unit; background replication compute is free.

## Slide 36 — Direct Lake: Import Speed, No Import Copy

- Power BI semantic-model storage mode that loads Delta tables from OneLake directly into memory.
- Refresh copies metadata/framing rather than full data.
- Loads only columns needed by the query.
- Reduces data latency and reuses Fabric investments.

## Slide 37 — OneLake Catalog: Explore, Govern, Secure

- Centralized place in Fabric to find, explore, and govern data.
- Embedded in Teams, Excel, and Copilot Studio.
- Explore: details, selectors, filters, object explorer.
- Govern: governance posture, sensitivity-label coverage, DLP coverage.
- Secure: workspace roles and OneLake security roles.
- Use OneLake Catalog for in-Fabric governance and Purview for enterprise-wide governance.

## Slide 38 — Migrating ADF & Synapse Pipelines to Fabric

- Opportunity to simplify governance, standardize patterns, and adopt CI/CD, OneLake integration, and Copilot.
- Paths:
  - Mount ADF in Fabric.
  - Use built-in upgrade experience.
  - Use PowerShell/manual rebuild.
- Component changes:
  - Mapping Data Flows → Dataflow Gen2.
  - Linked services → Connections.
  - Global parameters → Fabric Variable Library.
  - Integration Runtimes → Data Gateways.
